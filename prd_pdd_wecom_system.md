# 产品需求文档 (PRD) - PDD 派单与客勤管理系统 (MVP版)

## 1. 项目背景与目标
* **业务现状**：老板线上雇佣五湖四海的兼职客服和 PPT 设计师。客单价约几十块，日单量 60-100 单，日营业额约 3000 元。
* **核心痛点**：
    * 顾客沟通依赖 PDD 平台，多轮沟通效率低，容易漏单。
    * 手工加微信、建企微群拉设计师非常繁琐。
    * 兼职客服有“私自接单（飞单）”和“少报金额吃差价（虚报金额）”的风险。
    * 老板难以清晰掌控每个客服/设计师的工作负荷与产出，算提成困难。
* **业务目标**：打造一套开箱即用的“客服端浮窗 + 服务端 + 老板看板”系统，实现接单自动化、跨端协同、合规防飞单以及透明的账目和状态追踪。
* **技术栈期望**：采用大厂标准开发框架，保证稳定和规范。其中桌面端使用 **Go + Wails**，服务端采用 **Go (如 Gin/Kratos) or FastAPI**，管理端采用 **Vue3**。

│  ┌───────────────────────────────────────────────────────────┐  │

## 3. 核心功能模块 (MVP)

### 3.1 客服端：Go + Wails 桌面快捷浮窗
为兼职客服使用的电脑端软件，要求开箱即用（单个独立 .exe/APP，无环境依赖）。

* **身份绑定与设备认证**：
    * 首次运行输入激活码验证，绑定电脑 MAC 地址与工号，后续免密静默登录，防止代签到和代接单。
* **便捷录单面板**：
    * 提供表单项：PPT 主题、页数、交付时间、备注说明等。
* **智能防篡改（OCR 自动提取价格）**：
    * 客服直接上传 PDD 订单截图，浮窗上传至服务器，服务器通过国内大模型（通义千问VL/GLM-4V）进行 OCR，自动提取：**PDD 订单号** 与 **真实支付金额**。
    * 提取金额回填表单并锁定（不允许客服修改）。有效防飞单、防虚报吃回扣。
* **顾客建档**：
    * 由客服在 PDD 索要到的顾客联系方式（微信号 / 手机号）录入系统。
* **一键提交下单**：
    * 订单及数据推送到服务端，清空当前表单，通知后台后续流转。

### 3.2 服务端：订单大脑与企微机器人网关
所有状态流转与 API 集成由服务器在后台操作，保证安全性与高可用。

* **企业微信流程自动化 (核心价值)**：
    * **自动加好友**：根据客服录入的手机号/微信号，向顾客发送企微的添加好友邀请。
    * **抢单分发机制（防垄断与公平派单）**：
        1. 接到新单，服务端向所有状态“空闲”的设计师在企微内推送工单卡片通知包含基本信息。
        2. 设计师可在企微聊天窗口点击“立即接单”。
        3. 锁单逻辑：首位接单者锁定订单。
        4. 无人接单兜底：如果 5/10 分钟内未有人抢单，按排班或均匀分单策略强行指派一位设计师。
    * **自动建群拉人**：确立设计师后，企微通过接口，拉起包含【接单设计师、录单客服、系统企微机器人、顾客(通过微信互通)】的三/四方群。
    * **需求播报**：群建立后，企微机器人自动在群里发送一段格式良好的《PPT 设计需求清单》，确保双方信息绝对一致。
* **状态流转引擎**：
    * `待接单 (PENDING)` -> `已分配建群 (GROUP_CREATED)` -> `进行中 (DESIGNING)` -> `交付验收 (DELIVERED)` -> `已完结 (COMPLETED)`

### 3.3 管理端：老板专属 Vue3 数据看板
提供直观的数据透明度与员工绩效统计。

* **实时订单大厅**：
    * 滚动展示当前排队中的订单、各订单流转状态。
* **团队看板与考勤定位**：
    * 图表展示：当前客服、设计师的工作饱满度（例如：小李 3单进行中、小张 空闲）。可识别谁在恶意抢单却不产出。
* **财务与经营数据监控**：
    * 今日系统录入实际总金额（拦截了客服在中间拿差价的漏斗）、每单的 PDD 单号对应。
    * 营收曲线（按日/周/月报表，折线图、饼图）。

## 4. 后期可拓展规划 (Phase 2 & Phase 3)

系统基于大厂规范设计，未来可以低成本横向延伸以下环节：

1. **自动分润报表生成**：基于配置项直接计算每单中（平台扣点、设计师抽成、客服提成）净利归属，每月一键生成 Excel 发工资，解放财务手工算账。
2. **防盗版与水印系统**：交付物由平台小程序进行中转，支付尾款或顾客确认收货后，再释放无水印版本给顾客，防止顾客"骗图跑单"。
3. **交付超时监控与智能催更**：距离约定交付时间倒计时 3 小时，企微机器人私信设计师进行告警催更，避免退单。
4. **顾客满意度追踪**：订单标记完结时，机器人在群内发送问卷/评价链接，评分挂钩设计师考核绩效。

## 5. 项目落地技术约定（For 研发人员）

* **客服端 (Client)**：`Go 1.22+` / `Wails v2` / `Vue3 (Composition API) + TailwindCSS` (用于浮窗 UI 渲染)
* **服务端 (Server)**：`Go` (Gin 或 Kratos 框架) 确保并发与运行效率，提供 RESTful API，承接 OCR 模型调用。可选保留当前 `FastAPI` 搭配 Celery/RQ，如追求极致运行速度建议均用 Go 重写。
* **数据存储层 (DB)**：`PostgreSQL` 作为首选项（包含时区支持及良好的 JSONB 扩展），支持分润审计等强一致场景。
* **内部推送 (WeCom)**：全面打通企业微信“自建应用”能力，涵盖 API：获取通讯录成员、发送应用消息、创建客户群（`https://qyapi.weixin.qq.com/cgi-bin/externalcontact/groupchat/add`）、客户联系（`add_contact_way`）等。
* **OCR 与识别 (AI)**：集成 `阿里云百炼` / `智谱大模型平台` 的 Vision 模型。
