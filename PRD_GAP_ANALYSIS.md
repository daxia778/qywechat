## 四、差距项清单（按商业价值排序）

### P0 — 核心业务闭环缺口（直接影响营收或防损）

#### 1. 企微自动加好友
- **PRD 描述**: 客服录入顾客手机号/微信号后，服务端自动通过企微「客户联系」API 向顾客发送好友邀请
- **商业价值**: 这是 PRD 中标注的**"核心价值"**功能。当前建群流程需要人工加好友，大量手动操作抵消了自动化的效率提升。日均 60-100 单意味着每天要手动加 60-100 次好友
- **ROI 评估**:
  - **投入**: 约 2-3 天开发（对接企微 `add_contact_way` + `externalcontact` 系列 API）
  - **收益**: 每单节省 1-2 分钟人工操作 → 日均节省 1-3 小时客服人力
  - **ROI**: **极高** — 低投入、高频场景、直接提升效率
- **风险**: 需企微后台开通「客户联系」权限，API 调用频率有限制（每天不超过一定次数）
- **建议**: **立即开发**

#### 2. 顾客建档独立实体
- **PRD 描述**: 客服录入的顾客联系方式应形成独立档案，可追溯复购
- **商业价值**: 当前顾客信息仅作为订单字段存储，无法识别回头客、分析复购率、做客户分层运营
- **ROI 评估**:
  - **投入**: 约 1-2 天（新建 Customer 模型 + 关联 Order + 管理端页面）
  - **收益**: 识别复购客户可提升 LTV，为后续营销打基础
  - **ROI**: **高** — 低投入、长期价值
- **建议**: **下一迭代优先**

## 六、风险评估

### 高风险
1. **企微「客户联系」API 权限**: 自动加好友功能依赖企微后台开通「客户联系」能力，部分企业版本可能不支持或需额外审批。**建议立即确认企微后台权限状态**
2. **公网回调未就绪**: 企微回调 Token/EncodingAESKey 尚未配置（需公网域名），部分自动化流程在生产环境可能无法触发

### 中风险
3. **SQLite 并发瓶颈**: 日单量 60-100 当前可承受，但如果团队扩张或引入更多自动化写入（满意度评分、抢单日志等），写锁冲突概率上升
4. **Vue 遗留代码**: `admin-web/src/views/` 下仍有旧 Vue 文件，与当前 React 架构混合存在，增加维护复杂度

### 低风险
5. **技术栈偏差**: React 替代 Vue3 不影响功能实现，开发团队熟悉 React 即可
6. **CSV vs Excel**: 当前 CSV 导出可用，升级为 Excel 是体验优化而非阻断性问题

## 七、总结

MVP 在**核心派单流程**（录单 → OCR 防篡改 → 抢单分发 → 建群 → 需求播报 → 状态流转 → 超时催更）上实现度高，已具备投入生产的基本能力。

**最关键的缺口**是「企微自动加好友」— 这是 PRD 明确标注为核心价值的功能，也是当前流程中唯一需要大量人工介入的环节。日均 60-100 单 x 每单 1-2 分钟手动加好友 = 每天 1-3 小时纯人力浪费，补齐此功能的 ROI 最高。

其次建议关注「顾客建档」和「恶意抢单检测」，前者为后续运营分析奠定数据基础，后者直接解决管理效率痛点。

按推荐排期执行，**约 2-3 周可将 PRD 完成度从 74% 提升至 90%+**，剩余的防盗版水印和满意度追踪可视实际业务数据按需排期。
